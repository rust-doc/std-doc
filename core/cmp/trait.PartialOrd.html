Expand description

可以比较排序顺序的值的Trait。

Trait for values that can be compared for a sort-order.

可以分别使用 `<`、`<=`、`>` 和 `>=` 运算符调用此 trait 的 `lt`、`le`、`gt` 和 `ge` 方法。

The lt, le, gt, and ge methods of this trait can be called using the <, <=, >, and >= operators, respectively.

此 trait 的方法必须彼此一致，并且在以下意义上与 `PartialEq` 的方法一致：

The methods of this trait must be consistent with each other and with those of PartialEq in the following sense:

  • `a == b` 当且仅当 `partial_cmp(a, b) == Some(Equal)`。
  • a == b if and only if partial_cmp(a, b) == Some(Equal).
  • `a < b` 当且仅当 `partial_cmp(a, b) == Some(Less)` （由默认实现确保）。
  • a < b if and only if partial_cmp(a, b) == Some(Less) (ensured by the default implementation).
  • `a > b` 当且仅当 `partial_cmp(a, b) == Some(Greater)` （由默认实现确保）。
  • a > b if and only if partial_cmp(a, b) == Some(Greater) (ensured by the default implementation).
  • `a <= b` 当且仅当 `a < b || a == b`（由默认实现确保）。
  • a <= b if and only if a < b || a == b (ensured by the default implementation).
  • `a >= b` 当且仅当`a > b || a == b`（由默认实现确保）。
  • a >= b if and only if a > b || a == b (ensured by the default implementation).
  • `a != b` 当且仅当 `!(a == b)` （已经是 `PartialEq` 的一部分）。
  • a != b if and only if !(a == b) (already part of PartialEq).

如果 `Self` 和 `Rhs` 也实现了“Ord”，它也必须与 `partial_cmp` 一致（有关确切要求，请参阅该 trait 的文档）。 通过派生一些Trait并手动实现其他Trait，很容易意外地使他们不同意。

If Ord is also implemented for Self and Rhs, it must also be consistent with partial_cmp (see the documentation of that trait for the exact requirements). It’s easy to accidentally make them disagree by deriving some of the traits and manually implementing others.

对于所有 `a`、`b` 和 `c`，比较必须满足：

The comparison must satisfy, for all a, b and c:

  • 传递性：`a < b` 和 `b < c` 意味着 `a < c`。 `==` 和 `>` 都必须如此。
  • transitivity: a < b and b < c implies a < c. The same must hold for both == and >.
  • 对偶性：`a < b` 当且仅当`b > a`。
  • duality: a < b if and only if b > a.

请注意，这些要求意味着Trait本身必须对称且可传递地实现：如果 `T: PartialOrd` 和 `U: PartialOrd` 那么 `U: PartialOrd` 和 `T: PartialOrd`。

Note that these requirements mean that the trait itself must be implemented symmetrically and transitively: if T: PartialOrd<U> and U: PartialOrd<V> then U: PartialOrd<T> and T: PartialOrd<V>.

Corollaries

以下推论来自上述要求：

The following corollaries follow from the above requirements:

  • `<` 和 `>` 的非自反性：`!(a < a)`, `!(a > a)`
  • irreflexivity of < and >: !(a < a), !(a > a)
  • `>` 的传递性：如果 `a > b` 和 `b > c` 则 `a > c`
  • transitivity of >: if a > b and b > c then a > c
  • `partial_cmp` 的对偶性：`partial_cmp(a, b) == partial_cmp(b, a).map(Ordering::reverse)`
  • duality of partial_cmp: partial_cmp(a, b) == partial_cmp(b, a).map(Ordering::reverse)

Derivable

此 trait 可以与 `#[derive]` 一起使用。 当对结构进行派生时，它将根据结构成员的从上到下的声明顺序生成字典顺序。 当在枚举上“派生”时，变体按其从上到下的判别顺序排序。

This trait can be used with #[derive]. When derived on structs, it will produce a lexicographic ordering based on the top-to-bottom declaration order of the struct’s members. When derived on enums, variants are ordered by their top-to-bottom discriminant order.

How can I implement PartialOrd?

`PartialOrd` 只需要实现“partial_cmp”方法，其他方法由默认实现生成。

PartialOrd only requires implementation of the partial_cmp method, with the others generated from default implementations.

但是，对于没有总顺序的类型，仍然可以单独实现其他类型。 例如，对于浮点数，`NaN < 0 == false` 和 `NaN >= 0 == false`（参见 IEEE 754-2008 第 5.11 节）。

However it remains possible to implement the others separately for types which do not have a total order. For example, for floating point numbers, NaN < 0 == false and NaN >= 0 == false (cf. IEEE 754-2008 section 5.11).

`PartialOrd` 要求您的类型为“PartialEq”。

PartialOrd requires your type to be PartialEq.

如果您的类型是“Ord”，则可以使用“cmp”实现“partial_cmp”：

If your type is Ord, you can implement partial_cmp by using cmp:

use std::cmp::Ordering;

#[derive(Eq)]
struct Person {
    id: u32,
    name: String,
    height: u32,
}

impl PartialOrd for Person {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

impl Ord for Person {
    fn cmp(&self, other: &Self) -> Ordering {
        self.height.cmp(&other.height)
    }
}

impl PartialEq for Person {
    fn eq(&self, other: &Self) -> bool {
        self.height == other.height
    }
}
Run

您可能还会发现在类型的字段上使用“partial_cmp”很有用。 这是一个具有浮点 `height` 字段的 `Person` 类型示例，该字段是唯一用于排序的字段：

You may also find it useful to use partial_cmp on your type’s fields. Here is an example of Person types who have a floating-point height field that is the only field to be used for sorting:

use std::cmp::Ordering;

struct Person {
    id: u32,
    name: String,
    height: f64,
}

impl PartialOrd for Person {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        self.height.partial_cmp(&other.height)
    }
}

impl PartialEq for Person {
    fn eq(&self, other: &Self) -> bool {
        self.height == other.height
    }
}
Run

fn partial_cmp(&self, other: &Rhs) -> Option<Ordering>

如果存在，此方法返回 `self` 和 `other` 值之间的排序。

This method returns an ordering between self and other values if one exists.

Examples
use std::cmp::Ordering;

let result = 1.0.partial_cmp(&2.0);
assert_eq!(result, Some(Ordering::Less));

let result = 1.0.partial_cmp(&1.0);
assert_eq!(result, Some(Ordering::Equal));

let result = 2.0.partial_cmp(&1.0);
assert_eq!(result, Some(Ordering::Greater));
Run

当无法比较时：

When comparison is impossible:

let result = f64::NAN.partial_cmp(&1.0);
assert_eq!(result, None);

Implements comparison operations on strings.

Strings are compared lexicographically by their byte values. This compares Unicode code points based on their positions in the code charts. This is not necessarily the same as “alphabetical” order, which varies by language and locale. Comparing strings according to culturally-accepted standards requires locale-specific data that is outside the scope of the str type.
