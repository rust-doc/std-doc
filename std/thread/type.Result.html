线程的专用“结果”类型。

A specialized Result type for threads.

指示线程退出的方式。

Indicates the manner in which a thread exited.

`Result::Err` 变体中包含的值是线程panic的值； 也就是，调用 `panic!` 宏的参数。 与正常错误不同，此值不实现“错误”Trait。

The value contained in the Result::Err variant is the value the thread panicked with; that is, the argument the panic! macro was called with. Unlike with normal errors, this value doesn’t implement the Error trait.

因此，处理线程panic的明智方法是：

Thus, a sensible way to handle a thread panic is to either:

  1. 用“std::panic::resume_unwind”传播panic
  2. propagate the panic with std::panic::resume_unwind
  3. 或者如果线程打算成为一个子系统边界，应该隔离系统级故障，匹配 `Err` 变体并以适当的方式处理panic
  4. or in case the thread is intended to be a subsystem boundary that is supposed to isolate system-level failures, match on the Err variant and handle the panic in an appropriate way

一个在没有panic的情况下完成的线程被认为是成功退出。

A thread that completes without panicking is considered to exit successfully.

Examples

匹配加入线程的结果：

Matching on the result of a joined thread:

use std::{fs, thread, panic};

fn copy_in_thread() -> thread::Result<()> {
    thread::spawn(|| {
        fs::copy("foo.txt", "bar.txt").unwrap();
    }).join()
}

fn main() {
    match copy_in_thread() {
        Ok(_) => println!("copy succeeded"),
        Err(e) => panic::resume_unwind(e),
    }
}
