可以“未调整大小”为动态大小类型的类型。

Types that can be “unsized” to a dynamically-sized type.

例如，大小数组类型`[i8; 2]` 实现了 `Unsize<[i8]>` 和 `Unsize`。

For example, the sized array type [i8; 2] implements Unsize<[i8]> and Unsize<dyn fmt::Debug>.

`Unsize` 的所有实现都是由编译器自动提供的。 这些实现是：

All implementations of Unsize are provided automatically by the compiler. Those implementations are:

  • 数组`[T; N]` 实现 `Unsize<[T]>`。
  • Arrays [T; N] implement Unsize<[T]>.
  • 实现 trait `Trait` 的类型也实现了 `Unsize`。
  • Types implementing a trait Trait also implement Unsize<dyn Trait>.
  • 如果满足所有这些条件，则结构 `Foo<..., T, ...>` 实现 `Unsize>`：
  • Structs Foo<..., T, ...> implement Unsize<Foo<..., U, ...>> if all of these conditions are met:
    • `T: Unsize`。
    • T: Unsize<U>.
    • 只有 `Foo` 的最后一个字段具有涉及 `T` 的类型。
    • Only the last field of Foo has a type involving T.
    • `Bar: Unsize<>>`，其中 `Bar` 代表最后一个字段的实际类型。
    • Bar<T>: Unsize<Bar<U>>, where Bar<T> stands for the actual type of that last field.

`Unsize` 与 "ops::CoerceUnsized" 一起使用以允许 "用户定义" 容器，例如 "Rc" 包含动态大小的类型。 有关更多详细信息，请参阅“DST 强制 RFC”和“强制的 nomicon 条目”。

Unsize is used along with ops::CoerceUnsized to allow “user-defined” containers such as Rc to contain dynamically-sized types. See the DST coercion RFC and the nomicon entry on coercion for more details.
