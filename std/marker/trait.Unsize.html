可以“未调整大小”为动态大小类型的类型。

Types that can be “unsized” to a dynamically-sized type.

例如，大小数组类型 [i8; 2] 实现 Unsize<[i8]> 和 Unsize。

For example, the sized array type [i8; 2] implements Unsize<[i8]> and Unsize<dyn fmt::Debug>.

Unsize 的所有实现都是由编译器自动提供的。

All implementations of Unsize are provided automatically by the compiler.

Unsize 用于：

Unsize is implemented for:

  • [T; N] 是 Unsize<[T]>
  • [T; N] is Unsize<[T]>
  • T is Unsize 当 T: Trait
  • T is Unsize<dyn Trait> when T: Trait
  • Foo<..., T, ...> 是 Unsize> 如果：
  • Foo<..., T, ...> is Unsize<Foo<..., U, ...>> if:
    • T：取消大小
    • T: Unsize<U>
    • Foo 是一个结构
    • Foo is a struct
    • 只有 Foo 的最后一个字段的类型涉及 T
    • Only the last field of Foo has a type involving T
    • T 不属于任何其他字段的类型
    • T is not part of the type of any other fields
    • Bar：如果 Foo 的最后一个字段的类型为 Bar，则为 Unsize<>>
    • Bar<T>: Unsize<Bar<U>>, if the last field of Foo has type Bar<T>

Unsize 与 ops::CoerceUnsized 一起使用以允许“用户定义”容器（例如 Rc）包含动态大小的类型。 有关更多详细信息，请参阅 DST 强制 RFC 和有关强制的 nomicon 条目。

Unsize is used along with ops::CoerceUnsized to allow “user-defined” containers such as Rc to contain dynamically-sized types. See the DST coercion RFC and the nomicon entry on coercion for more details.
